# Universal Enchants

A Minecraft mod. Downloads can be found on [CurseForge](https://www.curseforge.com/members/fuzs_/projects) and [Modrinth](https://modrinth.com/user/Fuzs).

![](https://raw.githubusercontent.com/Fuzss/modresources/main/pages/data/universalenchants/banner.png)

## Configuring the mod
Universal Enchants allows you to define what enchantments can be applied to what items, and which enchantments are compatible with each other (meaning can be applied on a single item at the same time). This is done via individual `.json` config files (one per enchantment) found in `.minecraft/config/universalenchants`, sorted into sub-folders by namespace.

The configuration files are applied similar to a data pack, meaning they are loaded whenever a new world is, and can be reloaded while playing by running `/reload`.

The internal implementation of individual enchantments is rather complex and relies on a bunch of hard-coded special cases. All items present by default in the `items` and `anvil_items` fields are guaranteed to work, everything beyond that is untested and has a good chance of not working, especially modded enchantments.

### Config file structure
```
.minecraft/config/universalenchants/minecraft/smite.json
```
```json
{
  "items": [
    "#minecraft:enchantment_target/weapon",
    "#universalenchants:enchantment_target/axe",
    "#minecraft:enchantment_target/trident"
  ],
  "anvil_items": [
    "#minecraft:enchantment_target/weapon",
    "#universalenchants:enchantment_target/axe",
    "#minecraft:enchantment_target/trident"
  ],
  "incompatible": [
    "minecraft:bane_of_arthropods",
    "minecraft:impaling"
  ]
}
```

- `items`: The items this enchantment can be applied to in an enchanting table. This entry supports both single items and item tags (prefixed using `#`). Each kind of entry can be set as an inverse, to force it to be excluded instead of included (see [below](#excluding-entries-from-items-and-anvil_items)). When this list is empty, the enchantment cannot be applied to any item. If the list entry is missing though, vanilla compatibility rules for accepted items will be used.
- `anvil_items`: The items this enchantment can be applied to in an anvil. This will only rarely be different from `items`, an example is `minecraft:efficiency`. The exact same rules as for `items` apply.
- `incompatible`: Other enchantments that are incompatible with this enchantment (they cannot be applied together on a single item at the same time). An enchantment is always incompatible with itself, there is no setting for that. Setting an incompatibility for one of the two enchantments is enough for it to apply, meaning in reverse when removing an incompatibility it must be removed from the configs of both relevant enchantments.

### Excluding entries from `items` and `anvil_items`
Each entry (single enchantment or enchantment tag) can be inverted, allowing it to be excluded from other entries instead of included. This is useful when just a few values from a tag are not desired, to avoid having to include all contents of said tag manually minus the few supposed to be excluded.

To inverse an entry instead of
```json
{
  "items": [
    "minecraft:diamond_pickaxe"
  ]
}
```
you add
```json
{
  "items": [
    {
      "id": "minecraft:diamond_pickaxe",
      "exclude": true
    }
  ]
}
```
When `exclude` is set to false the entry will behave normally as without the `exclude` tag.

Alternatively it is possible to use a simplified syntax by prefixing an entry with an exclamation mark. E.g. `"minecraft:diamond_pickaxe"` turns into `"!minecraft:diamond_pickaxe"`, which has the exact same effect as setting the `exclude` field to `true`.

### Item tags for built-in enchantment categories
Vanilla internally uses hard-coded definitions for which enchantment can be applied to which items. Universal Enchants turns those definitions into dynamic tags which are generated in memory at runtime (meaning their contents will not show up in any file). This ensures items by other mods are automatically included in those tags, which the original vanilla definitions account for, too.

This table includes all the new item tags for enchantment compatibility dynamically generated by Universal Enchants based on vanilla's hard-coded definitions. When other mods are installed, additional tags can exist, also the listed tags can include additional items.

Default tag contents are mainly shared to be able to get a sense of the kind of items that are included.
